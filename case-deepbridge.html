DeepBridge

DeepBridge — текущий рабочий прототип AIOSE (AI OS Environment), среды для ИИ, которую я развиваю давно: локальный мост между браузерным DeepSeek и полноценным AI-агентом с инструментами. Ранний DeepGate был убран из-за неверного подхода — на смену пришёл более устойчивый DeepBridge: перехват и инжект запросов через браузер, управление сессиями, инструменты файловой системы и shell в Web UI.

Интерфейс
О проекте

DeepBridge — локальный мост, который превращает браузерный DeepSeek в полноценного AI-агента с доступом к файловой системе, терминалу и браузеру. Flask-сервер поднимается локально, Playwright управляет Chrome через CDP, а Web UI даёт интерфейс в стиле DeepSeek с поддержкой инструментов.

Эволюция подхода

AIOSE линия

AI OS Environment — обобщённое имя линии, среда для ИИ, развиваемая задолго до. DeepGate и DeepBridge — итерации внутри неё.

DeepGate убран

Первый подход к перехвату. Оказался неустойчивым и архитектурно неверным — снят с разработки.

DeepBridge текущий

Переосмысленный мост: более устойчивое управление сессиями и инжектом. Рабочий прототип, требует доработки.

Ключевые системы

CDP Bridge уникально

Управляет Chrome через Chrome DevTools Protocol. Позволяет перехватывать и инжектировать запросы в DeepSeek без официального API.

File System Tools

Агент читает, пишет и удаляет файлы в рабочей директории. Защита: path traversal проверка, whitelist директорий, protected files.

Shell Tools

Выполнение команд в терминале через агента. Whitelist-валидация команд, перехват stdout/stderr, защита от деструктивных операций.

Session Management

Управление чат-сессиями с историей, экспортом, контекстным меню и статистикой токенов. Полный жизненный цикл разговора.

Web UI в стиле DeepSeek

Минималистичный интерфейс с боковой панелью, режимами (Think / Normal), горячими клавишами, темами и поиском по истории. 20+ скриншотов UI-тестов.

Токен-авторизация

Постоянный токен доступа загружается из файла при рестарте. Защита от несанкционированного доступа к локальному серверу.
